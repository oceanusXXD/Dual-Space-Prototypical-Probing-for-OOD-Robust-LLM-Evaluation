# FLASK 最小验证实验设计

## 1. 目标

用一个小型 FLASK 子集完成三项验证：

1. Qwen3.5 Direct Judge 的直接评分能力；
2. 每个 `Domain × Task` cell 的 Qwen3.5 hidden-state 分类头评分能力；
3. 9 个 cell 分类头在完整 5×6 B-space 上的迁移表现。

数据使用单一 Domain prompts，并清理 `N/A`、空值和范围外分数。分类头使用
整数 1–5 的 Ground Truth。本 5×6 子集包含 550 个 prompts、8,250 条原始
候选回答；其中 8,236 条回答拥有至少一条有效的整数标签 B 行。再清理 10 条
`target_txt` 为空的回答后，实验实际使用 8,226 条非空 A-space 回答和 18,428 条
B-space 评分行。

## 3. A-space 与 B-space

每个入选的候选模型回答同时生成 A-space 和 B-space 数据。

| 空间 | 一条数据 | 唯一键 | 输入内容 |
|---|---|---|---|
| A-space | 一条候选回答 | `response_id` | `candidate_response` |
| B-space | 一条候选回答在一个当前 Skill 下的评分任务 | `response_id + task_id` | rubric、instruction、reference answer、candidate response |

同一个 B 行通过 `response_id` 关联到它的 A 行。Direct Judge 逐行读取 B-space
prompt 并产生 `direct_score`；该结果写回同一条 B 行。分类头也读取同一 B-space
prompt 的 hidden state。A-space 为每个被使用的 B 行保留对应的候选回答表征。

在本实验选择的单一 Domain 5×6 子集内：

| 空间 | 分区方式 | 分区数量 |
|---|---|---:|
| A-space | 按 Domain | 5 个 A-space 分区 |
| B-space | 按 `Domain × Task` | 30 个 B-space cell |

评分清理移除了 147 条 `N/A`、9 条空值、3 条范围外分数和 63 条非整数小数分数；
完整 5×6 子集保留 18,453 条整数 1–5 评分行。另有 10 条 `target_txt` 为空的候选
回答，对应 25 条有效整数评分行；这些行不进入模型输入。Direct Judge 与分类头共用
最终的 18,428 条 B 行及其关联的 8,226 条 A 行。

例如，一条回答进入 `Culture × Comprehension`：

```text
A: response_id = r_001
   candidate_response = 原始 target_txt

B: response_id = r_001
   task_id = Comprehension
   instruction = 原始 text
   reference_answer = 原始 answer
   rubric = Comprehension 的评分说明
   candidate_response = 原始 target_txt
   ground_truth = 当前 Skill 的 GPT-4 score
```

## 4. Direct Judge

Direct Judge 在完整 5×6 范围内执行，覆盖全部 18,428 条 B 行。每条 B 行同时
包含 Judge prompt、GPT-4 `ground_truth` 和 Qwen 的 `direct_score`。同批 B 行
抽取 Judge hidden state；每条 B 行关联到一条 A-space 回答，A-space 表征单独保存。

每条 B-space prompt 只执行一次 Qwen 推理：该次推理保存 B-space hidden state，
并生成 Direct Judge 的 `direct_score`。9 个分类头直接读取已保存的 B-space hidden
state，不执行第二次 B-space 推理。

### Direct Judge prompt

```text
You are an evaluator.

Skill and rubric:
{rubric}

Instruction:
{instruction}

Reference answer:
{reference_answer}

Candidate response:
{candidate_response}

Return one integer score from 1 to 5 in JSON format:
{"score": <1-5>}
```

`ground_truth` 与原始 GPT-4 `review` 仅用于推理后的指标计算。Domain 用于结果分组。

Direct Judge 报告全局结果及每个 `Domain × Task` cell 的 MAE、Exact Accuracy、±1 Accuracy 和 QWK。

## 5. 分类头 Judge

左上角 3×3 的每个 cell 训练一个独立的 5 分类头，共 9 个头：

```text
一个固定 Domain × Task cell 的 B-space prompt
        ↓ Qwen3.5
judge hidden state
        ↓
Linear 5-class head
        ↓
score = 1 / 2 / 3 / 4 / 5
```

9 个分类头及各自使用的数据：

| 分类头 | 训练与 ID 测试数据 |
|---|---|
| 1 | Humanities × Comprehension |
| 2 | Humanities × Factuality |
| 3 | Humanities × Logical Correctness |
| 4 | Language × Comprehension |
| 5 | Language × Factuality |
| 6 | Language × Logical Correctness |
| 7 | Social Science × Comprehension |
| 8 | Social Science × Factuality |
| 9 | Social Science × Logical Correctness |

每个 cell 先按 `question_id` 切分：10% 的题目组用于训练该 cell 的分类头，剩余
90% 的题目组用于该 head 的 ID 测试。抽取训练题目组时按分数分布分层，保证训练
集尽量覆盖 1–5 分。首轮使用预先固定的线性 5 分类头配置，不单独划分 validation。

| split | 每个 source cell 的比例 | 左上角 3×3 的预计 B 行数 |
|---|---:|---:|
| train | 10% | 约 658 |
| ID test | 90% | 约 5,919 |

训练使用 class-weighted cross entropy，以减轻 5 分标签较多带来的偏斜。

## 6. 分类头在完整 5×6 上的测试

每个分类头都预测完整 5×6 的 30 个 B-space cell。一次 Qwen hidden-state 抽取可被
9 个分类头重复使用，因此 9 个头不会增加 Qwen 特征抽取次数。

对于一个训练 head `H(D_source, T_source)`，目标 cell 的类型由下表定义：

| 目标 cell 与 source cell 的关系 | 测试类型 |
|---|---|
| `D_target = D_source` 且 `T_target = T_source` | ID test |
| `D_target ≠ D_source` 且 `T_target = T_source` | Domain shift |
| `D_target = D_source` 且 `T_target ≠ T_source` | Task shift |
| `D_target ≠ D_source` 且 `T_target ≠ T_source` | Joint shift |

因此，9 个 head 共产生 `9 × 30 = 270` 个 head–target-cell 结果。其中有 9 个
ID 结果、36 个 Domain shift 结果、45 个 Task shift 结果和 180 个 Joint shift
结果。每类结果分别报告 MAE、Exact Accuracy、±1 Accuracy 和 QWK。

source cell 的 ID test 使用该 cell 剩余 90% 题目组的全部有效 B 行。其余 target
cell 使用全部有效 B 行；对任意 head，测试 B 行的 `question_id` 不得出现在该
head 的训练题目组中。

## 7. 切分顺序

1. 保留单一 Domain 的 FLASK prompt，清理无效评分；
2. 为所有候选回答建立 A-space，为所有选中 Skill 建立唯一 B-space；
3. Direct Judge 在完整 5×6 的 18,428 条 B 行上推理，将 `direct_score` 写回
   对应 B 行，并在同一次推理中抽取 B-space Judge hidden state；
4. 为这些 B 行关联的 8,226 条 A 行抽取候选回答表征；
5. 左上角 3×3 的每个 source cell 按 `question_id` 分成 10% train 和 90% ID test；
6. 每个 head 在完整 5×6 测试 cell 上评测，并剔除该 head 已使用的训练 `question_id`。

同一 `question_id` 的 15 个候选模型回答及全部派生 B 行始终归入同一集合。

### 8.2.2 结果解读

Direct Judge 的直接评分能力明显更强：全体 B 行的 Exact 为 `0.5355`、MAE 为 `0.7839`、
QWK 为 `0.6820`。其中 Logical Correctness 是最稳定的 Skill（总体 Exact `0.7918`、
MAE `0.3601`、QWK `0.8762`）；Insightfulness 最困难（Exact `0.2643`、MAE `1.2762`、
QWK `0.3298`）。按 Domain 看，Humanities 的总体 Exact `0.4854`、QWK `0.5744` 最低；
History 的 QWK `0.7655` 最高，但该 Domain 仅有 916 行，不能只按单点数值下结论。

冻结 B-space 上训练的线性分类头没有超过 Direct Judge。即使在自身 source cell 的
ID test 上，九个头的 micro Exact 也只有 `0.3387`，QWK `0.0916`。在完全相同的
5,893 条 ID 测试行上，Direct Judge 的 Exact 为 `0.5922`、MAE 为 `0.6443`、QWK 为
`0.7434`，而分类头为 Exact `0.3387`、MAE `1.5184`、QWK `0.0916`；这说明仅用每个
source cell 的 10% 题目组训练的线性读出，尚未学到稳定的 1–5 有序评分边界。该比较
应理解为“Direct Judge 生成分数”与“冻结 Direct-Judge 表征上的小样本线性读出”两种
不同方法的对照，而不是在 Direct Judge 输出上继续叠加分类头。

从 ID 转移到 OOD 时，Domain shift 的退化最大：Exact 从 `0.3387` 降至 `0.2663`
（-7.24 个百分点），MAE 增加 `0.3113`，QWK 降至 `0.0242`。Task shift 也会使
Exact 降至 `0.2767`、QWK 降至 `0.0457`，但其 MAE `1.6466` 好于 Domain shift 的
`1.8297`；在当前设置下，领域改变比评分维度改变更破坏线性头的可迁移性。Joint shift
的 Exact `0.2560`、QWK `0.0335` 进一步说明两类变化不能被稳定处理。

头间差异很大：`H(Language, Logical Correctness)` 的 ID Exact `0.6143`、QWK `0.2209`，
切换 Domain 后仍有 Exact `0.5194`；`H(Humanities, Logical Correctness)` 在 Domain
shift 也保持 Exact `0.4629`，但切换 Task 时 QWK 变为 `-0.0048`。相反，
`H(Social Science, Comprehension)` 在 ID 上 Exact 仅 `0.1982`、QWK `-0.0397`，且
Domain shift 后继续下降。以上是描述性结果；10% question-group 训练、source cell
规模差异和标签不平衡都可能贡献于这些差异，需要单独增加训练预算或做多随机种子实验
才能归因。

### 8.2.3 合并 3×3、每 cell 15% 训练的共享分类头

作为小样本不足的对照，另一组实验不再训练九个独立 head，而是把左上 3×3 的 source
cell 混合为一个共享 5 分类头。按 `question_id` 做全局分组，并在每个 source cell 保留
约 15% 的题目组训练（小 cell 取最接近的整数）；同一个训练 `question_id` 从完整 5×6
测试范围全部排除。九个 source cell 共使用 976 条 B 行训练，source ID 留出集为 5,590
行，完整 5×6 测试集为 16,915 行。

| 方法 / 范围 | B 行数 | Exact | MAE | ±1 Accuracy | QWK |
|---|---:|---:|---:|---:|---:|
| 共享头：source train | 976 | 0.6096 | 0.7828 | 0.7971 | 0.5607 |
| 共享头：source ID 留出集 | 5,590 | 0.3089 | 1.4918 | 0.5698 | 0.1684 |
| Direct Judge：同一 source ID 留出集 | 5,590 | 0.5900 | 0.6447 | 0.8462 | 0.7448 |
| 共享头：完整 5×6 测试 | 16,915 | 0.2980 | 1.5576 | 0.5551 | 0.1172 |

相较于九个 10% 独立头的完整 5×6 微平均（Exact `0.2640`、MAE `1.7245`、QWK
`0.0373`），共享头的 QWK 提升到 `0.1172`，MAE 降至 `1.5576`；但两者同时改变了
head 共享方式与训练比例，不能将变化只归因于其中一个因素。共享头的 ID Exact 仍仅
`0.3089`，且同集 Direct Judge 为 `0.5900`，因此混合 3×3 数据能够改善有序一致性，
但尚不足以替代 Direct Judge。30 个 target cell 的完整结果见
`artifacts/flask_minimal_validation/pooled_3x3_15pct_head/cell_results.{csv,md}`。

### 8.2.4 MLP 与表征 debug

在同一份 15% pooled split 上，对仓库已有的 `SharedBackboneJudge` MLP 做了两个版本的
对照。MLP 使用可学习的两层混合、GELU、LayerNorm 和 validation-QWK 早停；加权版本
额外使用训练集 `class_weight="balanced"`。

| 头 | source ID Exact | source ID MAE | source ID QWK | 5×6 Exact | 5×6 MAE | 5×6 QWK |
|---|---:|---:|---:|---:|---:|---:|
| 线性（同 split） | 0.2936 | 1.8073 | 0.0667 | 0.2654 | 1.8393 | 0.0256 |
| 未加权 MLP | 0.4560 | 1.2424 | -0.0029 | 0.4115 | 1.3834 | -0.0024 |
| balanced MLP | 0.3449 | 1.3165 | -0.0479 | 0.3300 | 1.3804 | -0.0430 |

未加权 MLP 的 Exact/MAE 表面上变好，但它在 16,915 条 5×6 测试行中有 16,557 条
预测为 5，属于多数类坍缩；QWK 接近 0 说明并没有学到有序评分关系。加入 balanced
CE 后预测分布有所恢复，但 QWK 仍为负值，因此当前 MLP 不能直接替换线性头作为正式
评分器。

进一步固定线性头，只改变 B-space 表征，得到 source ID 的结果：

| 表征 | Ground Truth Exact | Ground Truth QWK | Direct-score Exact | Direct-score QWK |
|---|---:|---:|---:|---:|
| 第 23 层 masked-mean | 0.3585 | 0.1913 | 0.4986 | 0.2549 |
| 第 32 层 masked-mean（当前） | 0.3089 | 0.1684 | 0.4619 | 0.2203 |
| 两层 masked-mean | 0.3227 | 0.1977 | 0.4510 | 0.2205 |
| 两层 PCA-128 | 0.2948 | 0.1786 | 0.3932 | 0.1825 |

这表明 hidden state 并非完全没有评分信息：第 23 层比当前第 32 层更容易被线性读出，
而且对 Qwen 自己的 `direct_score` 也有一定可预测性。当前主要瓶颈更像是 masked-mean
把评分决策稀释了、取层不理想，以及小样本线性/MLP 读出没有稳定恢复 1–5 的有序关系，
而不是简单地把线性层换成更深的 MLP 就能解决。

后续若继续优化，优先级应是：先比较第 23 层、生成位置 hidden state 或更贴近评分
token 的 pooling；再使用 ordinal/cumulative loss 与 balanced weighting；最后才比较
不同 MLP 宽度和深度，并保持相同 question-group split 与多随机种子。

补充验证了一个单 cell 诊断：只在 `Humanities × Comprehension` 上，用已有 Direct Judge
completion 做 teacher forcing，并抽取两个更贴近最终打分的位置：score digit 前一 token
的 hidden（`pre_score_token`）和 score digit token 自身的 hidden（`score_token`）。两者
都只取第 23、32 层，仍然使用同一 question-group split、150 条训练 / 1,328 条测试。

| 表征 | Test Exact | Test MAE | Test ±1 | Test QWK |
|---|---:|---:|---:|---:|
| Direct Judge 原始分数 | 0.5459 | 0.7530 | 0.8110 | 0.6494 |
| prompt masked-mean hidden（原分类头表征） | 0.3825 | 1.1860 | 0.6860 | 0.1734 |
| pre-score-token hidden | 0.5354 | 0.6529 | 0.8569 | 0.6777 |
| score-token hidden | 0.5512 | 0.6732 | 0.8441 | 0.6810 |

这个结果说明分类头效果差的主因不是“hidden state 完全没信息”，而是当前 B-space
使用的 prompt masked-mean pooling 离真正的评分决策位置太远，平均后把 rubric、题目、
回答和格式 token 混在一起，评分信号被稀释。换到 score 附近的 hidden 后，QWK 从
`0.1734` 提升到约 `0.68`，已经略高于同 split 的 Direct Judge QWK；因此后续正式版本
应优先改成评分位置 hidden / pre-score logits 这一路，而不是继续在 masked-mean 表征上
堆 MLP。

### 8.3 评分指标定义

对任一结果行，记共有 `N` 条 B 行，第 `n` 条的 GPT-4 Ground Truth 为
`y_n ∈ {1, 2, 3, 4, 5}`，模型预测为 `ŷ_n ∈ {1, 2, 3, 4, 5}`。

| 指标 | 含义 | 计算公式 | 方向 |
|---|---|---|---|
| MAE | 平均相差多少分，1 分和 3 分的误差会按差值大小区分。 | `MAE = (1 / N) Σ[n=1..N] |ŷ_n - y_n|` | 越低越好 |
| Exact Accuracy | 预测与 Ground Truth 完全相同的比例。 | `Exact = (1 / N) Σ[n=1..N] 𝟙(ŷ_n = y_n)` | 越高越好 |
| ±1 Accuracy | 预测与 Ground Truth 相差不超过 1 分的比例。 | `±1 = (1 / N) Σ[n=1..N] 𝟙(|ŷ_n - y_n| ≤ 1)` | 越高越好 |
| QWK | 有序 1–5 评分的一致性。相差越远的预测受到越大惩罚，并扣除两组边际分布偶然一致的部分。 | `QWK = 1 - [Σ[i,j] w_ij O_ij] / [Σ[i,j] w_ij E_ij]` | 越高越好 |

QWK 的取值范围通常为 `[-1, 1]`，其中 `1` 表示完全一致，`0` 表示接近按边际分布
随机预测时的一致程度，负值表示一致性低于该基线。这里固定使用 5 个有序等级：

```text
O_ij = Ground Truth 为 i、预测为 j 的 B 行数量
n_i^y = Ground Truth 为 i 的 B 行数量
n_j^ŷ = 预测为 j 的 B 行数量
E_ij = n_i^y × n_j^ŷ / N
w_ij = (i - j)² / (5 - 1)² = (i - j)² / 16
i, j ∈ {1, 2, 3, 4, 5}
```

QWK 的分母为 0 时，该结果行的 QWK 记为不可计算并单独报告原因；MAE、Exact
Accuracy 和 ±1 Accuracy 仍照常计算。

## 9. B-space ViM OOD 测试

ViM 使用 B-space 的 Judge hidden state，检测“当前评分任务是否偏离某个分类头的
source cell”。Direct Judge 的 `direct_score` 只用于直接评分实验，不输入 ViM。

每个分类头 `H(D_source, T_source)` 独立拟合一个 ViM。source cell 留出的题目组
构成 ID；其余 29 个 target cell 构成 OOD。Domain shift、Task shift 和 Joint shift
沿用第 6 节的定义。

### 9.1 每个 source head 的数据 split

所有切分单位均为 `question_id`。同一题的 15 个候选回答及其全部 B 行始终进入同一
集合。下面的比例只针对一个 source cell。

| 子集 | 占 source cell 题目组的比例 | 用途 | 是否使用 Ground Truth |
|---|---:|---|---|
| `head_train` | 10% | 拟合特征预处理、训练 5 分类头、拟合 ViM reference | 使用 1–5 分训练分类头；ViM reference 只读取表征 |
| `vim_calibration` | 9% | 根据 ID 分数记录运行阈值 | 不使用 OOD 标签，不训练分类头 |
| `vim_id_test` | 81% | ViM 的 ID 评测样本 | 不参与训练、阈值或 rank 选择 |
| `vim_ood_test` | target cell 中排除 `head_train` 题目组后的全部 B 行 | ViM 的 OOD 评测样本 | 仅在最终计算 OOD 指标时使用预注册的 cell 归属 |

`head_train` 沿用第 5 节的按分数分布分层抽题方式。`vim_calibration` 从剩余 90%
题目组中按固定 seed 的 `question_id` 哈希抽取 10%，因此约占原 source cell 的 9%；
剩余约 81% 进入 `vim_id_test`。分类头的评分能力仍可在完整 90% 留出集上汇总；ViM
的 ID/OOD 指标只使用 `vim_id_test` 和 `vim_ood_test`。

对每个 head，`head_train` 的 `question_id` 从全部 target cell 移除。这样即使同一题
在另一个 Skill 下生成 B 行，也不会进入该 head 的 ViM OOD 测试。

### 9.2 ViM 拟合与测试顺序

每个 `H(D_source, T_source)` 按以下固定顺序执行：

1. 冻结该 head 的四个 `question_id` 集合：`head_train`、`vim_calibration`、`vim_id_test` 和各 target cell 的 `vim_ood_test`；
2. 仅在 `head_train` B 行上拟合特征预处理，并训练线性 5 分类头；
3. 用相同的训练 B 表征拟合 ViM reference。首轮固定 `rank = min(16, N_train - 2)`，其中 `N_train` 是该 head 的训练 B 行数；rank 不根据 OOD 测试结果调整；
4. 在 `vim_calibration` 的 ID B 行上计算 ViM 分数，并记录 source-ID 分位数阈值；
5. 在 `vim_id_test` 上计算 ID 分数，在每个 `vim_ood_test` cell 上计算 OOD 分数；
6. 每个 target cell 单独与同一个 `vim_id_test` 比较，生成 OOD 指标；
7. 按 ID、Domain shift、Task shift、Joint shift 及全部 target cell 汇总结果。

特征预处理、分类头、ViM reference、rank 和阈值均不能读取 target cell 的 Ground
Truth、OOD 分数或评测结果。若 `head_train` 的样本数不足以支持 `rank ≥ 1`，该 head
记为数据不足并单独报告，不从测试集补充样本。

### 9.3 ViM 评分与报告单位

首轮同时输出两种分数：

| 方法 | 输入 | OOD 分数 | 用途 |
|---|---|---|---|
| Residual-only ViM | 分类头输入的 B-space 表征 | 到 source 主子空间的 residual norm | 主方法；不依赖分类头 logits |
| Full ViM | 同一 B-space 表征、分类头 logits 和分类头权重 | virtual-logit OOD probability | 对照方法；检查 logits 是否提供额外信息 |

对一个固定 head，有 1 个 ID cell 与 29 个 OOD target cell。因此每种 ViM 方法生成
`9 × 29 = 261` 个“ID 对单个 OOD cell”的检测结果。每个结果的正类为 target cell 的
OOD B 行，负类为该 head 的 `vim_id_test` B 行。

| 报告层级 | 结果数量 | 指标 |
|---|---:|---|
| `source head × target cell` | 261 / ViM 方法 | AUROC、AUPR-OOD、FPR95、ID 阈值下的 ID 误报率和 OOD 检出率 |
| source-ID 参考 | 9 个 head | `vim_id_test` 的 ID 误报率、分数分位数和 B 行数 |
| 测试类型汇总 | Domain shift 36、Task shift 45、Joint shift 180 个结果 | 对相应 head–cell 结果做宏平均，并报告标准差 |
| 全部 ViM 总计 | 1 / ViM 方法 | 261 个 head–cell 结果的宏平均和标准差 |

AUROC、AUPR-OOD 和 FPR95 使用最终 ID/OOD 评测分数计算。运行阈值来自
`vim_calibration` 的 ID 分位数，仅用于报告部署式 ID 误报率和 OOD 检出率；它不参与
AUROC、AUPR-OOD 或 FPR95 的计算。

## 10. B-space 检测器对比测试

ViM 是本实验的重点检测器，同时需要与多类 OOD baseline 在完全相同的 head、表征和
数据 split 上比较。所有检测器复用 B-space hidden state 和已训练的线性分类头，不增加
Qwen 推理次数。

### 10.1 统一输入与统一 split

每个 `H(D_source, T_source)` 使用第 9.1 节的同一份数据：

```text
head_train       → 拟合预处理、分类头和需要 reference 的检测器
vim_calibration  → 校准每个检测器的 source-ID 阈值
vim_id_test      → 所有检测器共享的 ID 负类
vim_ood_test     → 所有检测器共享的 OOD 正类
```

每个检测器的输入为该 head 的预处理后 B-space 表征 `h`、5 分类 logits `z`，或两者。
`ground_truth` 只用于 Mahalanobis、RMD 的 source 类条件统计，以及最终的分类头训练；
target cell 的 Ground Truth 不参与 detector 拟合、参数设置或阈值校准。

### 10.2 检测器清单

首轮运行下列 15 个检测器。分数方向统一为“越大越 OOD”。

| 类别 | 检测器 | 使用的信息 | 是否在 `head_train` 拟合 |
|---|---|---|---|
| Logit | MSP | 最大 softmax 概率 | 否 |
| Logit | MaxLogit | 最大 logit | 否 |
| Logit | Energy | 全部 logits | 否 |
| Logit | GEN | logits 的广义熵分数 | 否 |
| Logit | GradNorm | 输出层梯度范数 | 否 |
| Logit prototype | KL Matching | source 预测概率原型 | 是 |
| Feature distance | Mahalanobis | source 中已出现的 score 类别中心与共享收缩协方差 | 是 |
| Feature distance | RMD | 类条件距离减去全局背景距离 | 是 |
| Feature distance | kNN | 与 source B 表征的最近邻距离 | 是 |
| Subspace | Residual-only ViM | source 主子空间外的 residual norm | 是 |
| Hybrid | Full ViM | residual 与分类头 logits | 是 |
| Feature/logit post-hoc | ODIN | 分类头输入表征上的扰动后置信度 | 否 |
| Feature/logit post-hoc | ReAct | source 分位数截断后的 Energy | 是 |
| Feature/logit post-hoc | DICE | source 贡献掩码后的 Energy | 是 |
| Feature/logit post-hoc | ASH-B | 激活重整后的 Energy | 否 |

ODIN 在冻结的分类头输入表征上实施扰动；它不是对原始文本重新求梯度。ReAct、DICE、
ASH-B、GEN、KL Matching 和 GradNorm 按本项目的机制实现运行，结果标注为
“OpenOOD-style post-hoc”对照，不宣称与其他框架的数值完全一致。

### 10.3 预先固定的参数

以下参数在正式 target OOD 测试前冻结。它们不能根据 5×6 的 OOD 结果修改。

| 检测器 | 固定参数 |
|---|---|
| MSP、MaxLogit | 无额外参数 |
| Energy | temperature = `1` |
| GEN | `gamma = 0.10` |
| KL Matching、GradNorm | 无额外参数 |
| Mahalanobis、RMD | Ledoit–Wolf 收缩协方差，ridge = `1e-5` |
| kNN | `k = min(10, N_train - 1)`，Euclidean distance，L2 normalization |
| Residual-only ViM、Full ViM | `rank = min(16, N_train - 2)` |
| ODIN | temperature = `1000`，feature-space epsilon = `0.0014` |
| ReAct | source feature 0.90 分位数截断 |
| DICE | source contribution sparsity = `0.90` |
| ASH-B | percentile = `65` |
| 所有检测器 | `vim_calibration` 的 0.90 / 0.95 source-ID 分位数作为 soft / hard 运行阈值 |

若某个 detector 因样本数、数值异常或类别统计无法拟合，则该 detector–head 组合记录
为 `not_applicable`，保留失败原因；不得换用 target cell 数据完成拟合。

### 10.4 逐 detector 测试顺序

对每一个 source head，按以下顺序处理全部 15 个检测器：

1. 用 `head_train` 拟合预处理和分类头，并保存分类头的权重、bias、类别覆盖情况；
2. 用 `head_train` 拟合所有需要 source reference 的 detector；
3. 在 `vim_calibration` 计算每个 detector 的 source-ID 分数，分别保存 0.90 / 0.95 阈值；
4. 在 `vim_id_test` 计算该 detector 的 ID 分数；
5. 依次在 29 个 `vim_ood_test` target cell 计算 OOD 分数；
6. 每个 target cell 与同一份 `vim_id_test` 组成一次二元 ID/OOD 评测；
7. 保存逐 B 行分数、逐 head–cell 指标和按 shift type 的汇总结果；
8. 所有基础 detector 结果固定后，才允许单独开展 A/B 融合实验。

每个 detector 有 `9 × 29 = 261` 个 ID 对 OOD cell 结果。完整首轮共有
`15 × 261 = 3,915` 个 detector–head–cell 指标行；这些计算只使用已缓存表征，主要
在 CPU 上完成。

### 10.6 OOD 指标解释

| 指标 | 含义 | 方向 |
|---|---|---|
| AUROC | 随机抽取一条 ID B 行和一条 OOD B 行时，OOD 分数更高的概率 | 越高越好 |
| AUPR-OOD | 将 OOD 视为正类后的 precision–recall 曲线面积 | 越高越好 |
| FPR95 | OOD 召回率达到 95% 时的 ID 误报率 | 越低越好 |
| ID FPR@0.95 阈值 | 用 source calibration 的 0.95 分位数作阈值时，ID test 被报警的比例 | 越低越好 |
| OOD TPR@0.95 阈值 | 用同一 source-ID 阈值时，target OOD 被报警的比例 | 越高越好 |

AUROC、AUPR-OOD 和 FPR95 是 detector 排序质量指标；阈值指标描述部署时采用
source-only 阈值后的报警行为。二者分开报告，避免使用 target OOD 标签选择运行阈值。
