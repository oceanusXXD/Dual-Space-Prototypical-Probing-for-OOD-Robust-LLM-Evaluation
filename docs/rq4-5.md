## RQ4：融合置信度能否准确预测判官失效，并提供风险保证？

### 研究问题

**RQ4：ViM 认知新颖度、监督误差头和集成分歧组成的融合置信度，是否显著优于单一 ViM？它能否识别 ViM 无法发现的 hard-ID 样本，并通过 conformal risk control 控制高置信样本的实际评分风险？**

RQ2 已经证明 ViM 能够识别一定程度的分布变化，并且高 ViM 样本的平均评分误差通常更大。但是，ViM 与单样本绝对误差的相关系数只有约 0.07–0.08，说明单独使用分布距离不足以准确判断判官是否会出错。

因此，RQ4 不再只使用 ViM，而是构造三类互补信号：

1. **ViM residual score**：判断样本是否偏离训练表征分布；
2. **监督误差头 `g(h)`**：预测判官在该样本上是否容易评分错误，主要识别 hard-ID；
3. **集成分歧或无标签准确率估计**：从模型之间的一致性判断当前批次的整体可靠性。

三类信号融合后形成最终 risk/confidence score，再使用 conformal risk control 在 calibration set 上确定阈值。

### 需要完成的实验

#### 1. 单样本错误预测

定义两种错误：

* Exact error：预测分数不等于真实分数；
* Large error：预测分数与真实分数相差超过 1。

| Model | Risk signal       | Error target | AUROC ↑ | AUPR ↑ | AURC ↓ | Top-20% risk error | Bottom-20% risk error |
| ----- | ----------------- | ------------ | ------: | -----: | -----: | -----------------: | --------------------: |
| 0.8B  | ViM only          | Large error  |         |        |        |                    |                       |
| 0.8B  | Head confidence   | Large error  |         |        |        |                    |                       |
| 0.8B  | Error head `g(h)` | Large error  |         |        |        |                    |                       |
| 0.8B  | ViM + `g(h)`      | Large error  |         |        |        |                    |                       |
| 0.8B  | Three-leg fusion  | Large error  |         |        |        |                    |                       |
| 4B    | ViM only          | Large error  |         |        |        |                    |                       |
| 4B    | Three-leg fusion  | Large error  |         |        |        |                    |                       |

这一实验回答融合置信度是否比 ViM 更准确地预测判官错误。

#### 2. hard-ID 与 benign-OOD 分析

| Sample type | Distribution | Judge result | ViM expected | Error head expected |
| ----------- | ------------ | ------------ | ------------ | ------------------- |
| Easy-ID     | ID           | Correct      | Low risk     | Low risk            |
| Hard-ID     | ID           | Wrong        | Low risk     | High risk           |
| Benign-OOD  | OOD          | Correct      | High risk    | Low risk            |
| Harmful-OOD | OOD          | Wrong        | High risk    | High risk           |

重点报告：

* ViM 漏掉多少 hard-ID；
* `g(h)` 能找回多少 hard-ID；
* 融合后是否减少 benign-OOD 的错误报警；
* harmful-OOD 是否集中在最终高风险区域。

#### 3. selective scoring

按照融合风险从高到低拒绝样本，报告剩余样本的评分质量：

| Method            | Coverage | MAE ↓ | Large-error rate ↓ | QWK ↑ |
| ----------------- | -------: | ----: | -----------------: | ----: |
| No rejection      |     100% |       |                    |       |
| ViM only          |      90% |       |                    |       |
| ViM only          |      80% |       |                    |       |
| Fusion confidence |      90% |       |                    |       |
| Fusion confidence |      80% |       |                    |       |

如果融合方法有效，随着高风险样本被拒绝，剩余样本的 MAE 和错误率应明显下降，同时保持较高 coverage。

#### 4. conformal 风险控制

在独立 calibration set 上确定置信度阈值，验证：

> 被系统接受的高置信样本中，Large-error rate 是否不超过预设的 α。

| Model | Confidence method | Target risk α | Coverage ↑ | Empirical risk ↓ | Risk CI upper bound | Guarantee pass |
| ----- | ----------------- | ------------: | ---------: | ---------------: | ------------------: | -------------- |
| 0.8B  | ViM only          |          0.10 |            |                  |                     |                |
| 0.8B  | Fusion confidence |          0.10 |            |                  |                     |                |
| 4B    | Fusion confidence |          0.10 |            |                  |                     |                |

还需要分别报告：

* ID calibration → ID test；
* ID calibration → Domain shift；
* ID calibration → Task shift；
* ID calibration → Joint shift；
* weighted conformal 是否能在 covariate shift 下改善风险控制。

### RQ4 当前状态

已完成 0.8B strict final prelogit B-space 的全量 RQ4 实验：

* 脚本：`scripts/llm_judge_ood/65_run_flask_rq4_conformal_failure_risk.py`
* 输出目录：`artifacts/flask_minimal_validation/rq4_conformal_08b_5x6_prelogit`
* 覆盖：9 个 3x3 source heads，完整 5x6 target cells，主风险为 `pm1_error`
* 风险分数：`residual_vim`、`mahalanobis`、`rmd`、`knn`、`logit_uncertainty`、`g_error_probability`、`fusion_equal_ecdf`、`fusion_guarded_ecdf`
* conformal：source calibration 上选择最大接受阈值 `s <= t`，要求 accepted calibration loss 的 Clopper-Pearson UCB ≤ alpha，并对候选阈值做 Bonferroni delta spending
* weighted conformal：每个 target cell 用 score-feature logistic density ratio 估计 `p_target / p_cal`，权重 clip=10，并用 effective-n beta UCB 做保守阈值选择

实跑结论：

* alpha=0.10 和 alpha=0.20 下，pm1 large-error 风险目标过严，所有 source/weighted conformal 阈值都无法得到非零 coverage；这是一个有效负结果，说明当前 head 的 source calibration large-error 基线已高于该风险预算。
* alpha=0.30 下开始可行：target-cell 结果中 399 个评估条目得到非零阈值，平均 accepted coverage 约 0.782。
* 单样本错误预测上，`logit_uncertainty` 和 `fusion_equal_ecdf` 是最强信号之一；最佳 all-target AUROC 约 0.642（`H(Humanities, Logical Correctness)` 的 `logit_uncertainty`），对应 fusion 约 0.638。
* hard-ID 诊断显示单一 ViM 会漏掉低 ViM 但判官错误的样本；在 alpha=0.30 示例中，`residual_vim` 对 hard-ID low-ViM wrong 的 reject rate 可为 0，而 `logit_uncertainty` 可达到约 0.60。
* benign-OOD 诊断显示纯 novelty 信号会把高 ViM 但判官正确的 OOD 样本大量拒绝；fusion/logit 信号能显著降低该类误报。

4B 尚未完成全量 RQ4：当前只有局部 3x3 pre-score/prelogit artifact，缺少完整 5x6 的有效 pre-score/prelogit B-space，因此不能用已删除的 mean B-space 结果补齐。

### RQ4 最终回答模板

> 三条腿融合的置信度在单样本错误预测、hard-ID 识别和 selective scoring 上显著优于单独的 ViM。误差头能够补充 ViM 无法识别的分布内难样本，ViM 则提供未见分布的新颖度信息。经过 conformal risk control 标定后，系统能够在保持一定 coverage 的同时，将被接受评分的实际错误率控制在预设风险水平以内。

如果实验没有达到该结果，则应写成部分支持，并明确哪一条腿或哪一种 shift 下失效。

## RQ5：低置信触发后，轻量更新能否安全修复 far-shift？

### 研究问题

**RQ5：当融合置信度判断判官在新分布上不可靠时，head-only、双层对齐和无标签测试时自适应能否恢复评分性能？更新后的模型能否通过 Gate，避免破坏原有 source 性能？**

RQ5 的核心不是简单重新训练分类头，而是验证不同级别更新方法对 near-shift 和 far-shift 的修复能力。

### 更新阶梯

| Level | Method                    | Updated part              |   Labels |
| ----- | ------------------------- | ------------------------- | -------: |
| L0    | Head-only re-training     | Linear head               | Few-shot |
| L1    | Dual-layer alignment      | Alignment map `T`         | Few-shot |
| L2    | TENT-style adaptation     | Affine scale/shift in `T` |     Zero |
| L3    | Strongly regularized LoRA | Top LLM layers            | Few-shot |

其中 L1 的形式为：

`prediction = H_s(T(h))`

* 冻结原始 source head `H_s`；
* 冻结 Qwen；
* 只训练轻量映射 `T`；
* 使用 source replay、anchor loss 和 residual-geometry regularization 防止破坏原有 OOD 几何。
