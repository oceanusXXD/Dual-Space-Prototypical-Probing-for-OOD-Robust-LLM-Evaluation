## RQ1：冻结 LLM 表征能否支撑 ID 评分？

### RQ1a：ID 上是否优于 Direct Judge？

**结论：对 0.8B Direct Judge，是的；对 4B Direct Judge，不是。**

在 0.8B 上，Direct Judge 的数字输出虽然 parse rate 尚可，但评分质量很弱：full 10×12 的 MAE = 2.604、Exact = 14.1%、±1 Acc = 27.0%、QWK = -0.007。相同 5×6 评估范围内，冻结 strict-prelogit B-space 上训练的线性头显著优于 0.8B Direct Judge，尤其是 MAE 和 QWK。

| Model / scope | Method | Test rows | MAE ↓ | Exact ↑ | ±1 Acc ↑ | QWK ↑ | Notes |
| --- | --- | ---: | ---: | ---: | ---: | ---: | --- |
| 0.8B full 10×12 | Direct Judge | 75,619 | 2.604 | 14.1% | 27.0% | -0.007 | full B-space direct result |
| 0.8B 5×6 ID | Direct Judge, same parsed rows | 5,411 | 2.691 | 14.2% | 25.0% | 0.007 | same rows as head where direct score parsed |
| 0.8B 5×6 ID | Majority baseline per source cell | 5,893 | 1.252 | 46.6% | 66.6% | 0.000 | strong exact because labels are skewed |
| 0.8B 5×6 ID | Cell linear head, 3×3 source heads | 5,893 | 1.196 | 43.1% | 68.4% | 0.230 | improves MAE / ±1 / QWK over majority |
| 0.8B 5×6 ID | Pooled 3×3 linear head, 15% questions | 5,590 | 1.056 | 41.1% | 69.6% | 0.429 | best 0.8B ID head result |
| 0.8B 5×6 ID | Pooled 3×3 MLP | 5,590 | 1.195 | 44.6% | 67.4% | 0.311 | worse than pooled linear on MAE/QWK |
| 4B 3×3 ID | Direct Judge, same splits | 5,893 | 0.644 | 59.2% | 84.7% | 0.758 | strong Direct Judge baseline |
| 4B 3×3 ID | Pre-score-token linear head | 5,893 | 0.689 | 54.3% | 85.4% | 0.640 | does not beat 4B Direct Judge |
| 4B 3×3 ID | Score-token linear head | 5,893 | 0.667 | 54.7% | 85.6% | 0.686 | closer, but still below Direct Judge QWK |

Interpretation：

- 0.8B 的 Direct Judge 本身不可靠；冻结 B-space head 相当于把“模型内部评分表征”重新校准到 FLASK label，因此 MAE 从 2.69 降到约 1.21，QWK 从约 0.01 提高到约 0.24。
- 0.8B majority baseline 的 Exact 很高，说明 FLASK cell 内 label 分布偏斜；因此 Exact 不能单独作为主结论，MAE 和 QWK 更能反映 ordinal scoring quality。
- Pooled 3×3 linear head 是当前 0.8B 最强 ID 配置：MAE = 1.056，QWK = 0.429。
- 4B Direct Judge 已经很强，冻结 score-token / pre-score-token head 只能接近 Direct Judge，不能声称优于 Direct Judge。

## RQ2：hidden-state detector 能否识别 OOD？

### RQ2a：总体 detector 排名

**结论：B-space detector 有中等 OOD 信号，但不是强检测器；kNN、Mahalanobis、Residual-only ViM 处于同一档，Full ViM 明显更差。**

0.8B 5×6 detector suite 的宏平均结果如下。

| 0.8B 5×6 | kNN | 261 | 0.664 | 0.570 | 0.683 | 0.084 | 0.153 |

| 0.8B 5×6 | Energy | 261 | 0.521 | 0.511 | 0.874 | 0.092 | 0.174 |
| 0.8B 5×6 | MSP | 261 | 0.493 | 0.487 | 0.915 | 0.091 | 0.106 |

Interpretation：

- 0.8B B-space 的 detector 信号是可见的，但 AUROC 最高也只有约 0.66，FPR95 仍高达约 0.68。
- full 10×12 的 residual-only ViM AUROC 与 5×6 接近（0.658 vs. 0.654），但 AUPR-OOD 更低、hard-threshold OOD TPR 只有 9.8%，说明扩大 cell 覆盖没有自动解决部署阈值问题。
- kNN 是当前宏 AUROC 最强方法，但只比 Mahalanobis / Residual-only ViM 高约 0.006–0.010，不能声称压倒性优势。
- Full ViM 的 hard threshold 行为很差，ID FPR 达 33.9%，说明把 classifier origin / logits 等完整 ViM 分量直接用于该 setting 不如 residual-only geometry 稳定。
- 4B 3×3 basic ViM smoke 更弱，不能支持“更大模型 detector 更强”的结论。

### RQ2c：ViM OOD 分数与评分误差的关系

这个新增实验直接回答：“ViM 得到的 OOD 分数越高时，Judge head 的 MAE 和 QWK 如何变化？”做法是对每个 3×3 source cell head，先用该 head 的 source training rows 拟合 residual-only ViM，再把所有非训练 B-space rows routed through 该 head，按 held-out source-ID calibration rows 得到的 ViM percentile 分桶，最后在每个桶内统计 prediction vs label 的 MAE、Exact、±1 Acc 和 QWK。

| ViM calibration bin | Routed eval rows | MAE ↓ | Exact ↑ | ±1 Acc ↑ | QWK ↑ | Mean ViM percentile |
| --- | ---: | ---: | ---: | ---: | ---: | ---: |
| p00_p50 | 38,507 | 1.184 | 35.9% | 68.7% | 0.166 | 0.262 |
| p50_p75 | 44,477 | 1.277 | 36.5% | 64.7% | 0.176 | 0.618 |
| p75_p90 | 37,304 | 1.424 | 34.4% | 60.3% | 0.131 | 0.829 |
| p90_p95 | 28,889 | 1.441 | 29.7% | 59.4% | 0.178 | 0.932 |
| p95_p100 | 16,002 | 1.468 | 31.5% | 57.9% | 0.130 | 0.984 |

Within-head decile 分桶给出更直接的高低端对比：最低 decile 的 MAE = 1.130、QWK = 0.221；最高 decile 的 MAE = 1.589、QWK = 0.035。整体 association 是显著但偏弱的：Spearman(raw ViM score, absolute error) rho = 0.071；Spearman(calibrated ViM percentile, absolute error) rho = 0.077。最高 calibration band 相对最低 band 的 MAE 增加 +0.284，QWK 下降 -0.036。

按 shift type 拆分后，MAE 上升趋势仍然存在，但 QWK 不完全单调：

| Shift type | Low ViM bin MAE / QWK | High ViM bin MAE / QWK | Direction |
| --- | ---: | ---: | --- |
| ID test | 1.119 / 0.255 | 1.396 / 0.143 | MAE worsens; QWK drops |
| Domain shift | 1.250 / 0.162 | 1.391 / 0.184 | MAE worsens; QWK non-monotonic |
| Task shift | 1.083 / 0.216 | 1.412 / 0.147 | MAE worsens; QWK drops |
| Joint shift | 1.198 / 0.138 | 1.501 / 0.115 | MAE worsens; QWK drops |

Interpretation：

- ViM OOD score 可以作为 **error-risk ranking signal**：分数越高，平均绝对误差整体越大，最高分区间的 ±1 accuracy 也明显更低。
- 这个 signal 不够强，不能单独作为“高分必错”的判别器；rho 只有约 0.08，说明它更适合作为 risk stratification / gating feature，而不是替代 head confidence。
- QWK 在最高分区间通常下降，但中间分桶和 Domain shift 上不严格单调；论文中应写成“high ViM score is associated with worse scoring quality”，不要写成严格单调定律。

- 除 GradNorm 方向相反外，其余 14 个 detector 的 sample-level association 都是正向且显著；但 165,179 条 routed records 会让很小的效应也得到极小 p-value，而且同一原始样本会经过多个 heads、记录并非完全独立。因此 p-value 只作描述性证据，应优先比较 rho 和高低桶差值。

- Residual ViM、RMD、Mahalanobis、kNN 的 OOD AUROC 约 0.65–0.66，在 RQ2a 中最擅长区分 distribution shift；但它们与单条 Judge error 的 rho 只有 0.078–0.088。这说明 **shift detection quality 和 error prediction quality 是两个不同目标**。
- Full ViM 对 error risk 的排序强于 residual-only ViM（0.102 vs. 0.078），但它的 OOD AUROC 更弱、ID hard-threshold FPR 更高；不能据此称 Full ViM 是更好的通用 OOD detector。
- 分 shift 后，几何方法仍然只有弱 error association，因此总体弱相关不是由混合不同 shift 导致的 Simpson effect。Domain 内部最弱（rho 约 0.04–0.06），Task 内部相对最强（约 0.08–0.10）。
- Domain 的几何 OOD score 能检测表征分布已经变化，但 group QWK rho 接近 0；Task shift 会直接改变评分 criterion，因此 detector score 与 MAE/QWK degradation 的关系更稳定。极端桶仍有差异，说明这些方法可做粗粒度 risk stratification，但无法提供稳定的逐级单调排序。
- DICE 的 D1→D10 端点明显恶化，但 90 个 head-decile group 的 QWK rho 不显著（rho = 0.065, p = 0.544）；ASH-B 也没有稳定的 QWK 单调关系（rho = -0.046, p = 0.668）。
- GradNorm 在该线性 Judge setting 中方向反转：高分桶反而 MAE 更低、QWK 更高，因此不能直接按当前“高分 = OOD risk”方向用于 error gating。
- MMD 没有放进这张表，因为它是 window-level two-sample drift statistic，不为单条 prediction 产生天然的 OOD score；它与窗口性能的关系属于 RQ3，而不是这里的 sample-level correlation。

当前 error-risk gating 的最合理候选是 ReAct / MaxLogit / Energy，几何 detector 则仍适合承担 distribution-shift monitoring。论文中不应把“最佳 OOD detector”和“最佳 Judge failure predictor”写成同一个排名。

### RQ3 结论

当前 RQ3 应写成否定或未充分支持：

> In the current FLASK 0.8B 5×6 window experiment, scalar drift statistics fire too often under ID-only streams, while the calibrated B-MMD residual-vector test is overly conservative and shows no persistent detection power. Therefore, the current artifacts do not yet support a deployable early-warning claim.

要把 RQ3 改成正向结论，需要补跑：

- window size = 50 / 100 / 200；
- OOD ratio = 0 / 5 / 10 / 20 / 50 / 100%；
- at least 100–200 replay episodes per condition；
- 1 / 2 / 3 consecutive-window rules；
- A-MMD、B-MMD、hard-OOD rate、mean residual score 的统一对比；
- macro CI or seed-level std。

## 可直接写入论文的简短结论

For RQ1, frozen Judge representations contain usable ordinal scoring information. On the 0.8B model, a linear probe trained on strict final-prelogit B-space substantially improves over direct digit judging on the same rows, reducing MAE from 2.69 to 1.21 on ID and maintaining lower MAE under Domain, Task, and Joint shifts. However, the effect is model-dependent: for the 4B model, Direct Judge remains stronger than score-token probes.

For RQ2, B-space detectors provide only moderate OOD separability. kNN, Mahalanobis, RMD, and Residual-only ViM cluster around AUROC 0.65–0.66 on the 0.8B 5×6 setting, with Joint shifts easier than Domain shifts. The high FPR95 values show that ranking signal exists but thresholded deployment remains weak. A separate 15-detector error-risk analysis shows that the OOD ranking and Judge-failure ranking objectives differ: ReAct, MaxLogit, and Energy have the strongest sample-level associations with absolute scoring error (Spearman rho around 0.19), whereas the geometric detectors that lead OOD AUROC have weaker error associations around 0.08. Across within-head risk deciles, ReAct increases MAE from 0.852 to 1.621 and reduces QWK from 0.341 to 0.041. Current artifacts do not yet establish A-space / B-space complementarity because A-space detector results are not present.

For RQ3, the current monitoring configuration should not be claimed as deployable. B-MMD is well-calibrated on ID-only windows but has zero persistent detection in the tested replay setup, while hard-OOD rate and mean residual score detect some drift only at the cost of unacceptable ID false alarms.
